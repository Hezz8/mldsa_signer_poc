\chapter{ML-DSA AXI Wrapper Specification}

\section*{Purpose}
This chapter specifies the stable PoC AXI-Lite wrapper that presents the hardware signing engine to Linux software.

\section{Design Intent}
The wrapper provides a narrow, deterministic control and status surface between PS software and PL logic. It remains intentionally simple so that software, simulation, and future core integration can converge on a stable contract.

\section{Current Implemented Structure}
The repository now contains a stable AXI-Lite wrapper that instantiates a generic ML-DSA engine adapter. The wrapper owns the PS-visible register map and sequencing contract, while the adapter owns the downstream engine-facing completion behavior.

\section{Primary Functions}
\begin{itemize}[leftmargin=*]
  \item Accept a 64-byte digest written by software.
  \item Expose control bits for start and clear-status behavior.
  \item Report idle, busy, done, and error status.
  \item Expose signature length and a read window for signature data.
  \item Remain agnostic to whether the adapter is operating in `STUB` or `CORE\_PLACEHOLDER` mode.
\end{itemize}

\section{Wrapper-to-Adapter Boundary}
The wrapper launches the adapter with a `start` event and a 64-byte digest, then waits for adapter completion. The adapter returns busy, done, error, signature length, and signature buffer information. The wrapper does not encode knowledge of future ML-DSA-OSH internal state machines or datapath topology.

\section{Current Behavioral Model}
The wrapper operates as a single-request engine for the current phase. Software shall not issue a new `start` command while the wrapper reports busy. A valid `start` clears stale visible results, launches the adapter, and later exposes a deterministic result once the adapter reports completion.

\section{Current Adapter Modes}
\begin{itemize}[leftmargin=*]
  \item \textbf{STUB:} fixed 128-byte signature, deterministic two-cycle completion latency, and signature content `STUBSIG || digest || zero padding`.
  \item \textbf{CORE\_PLACEHOLDER:} fixed 128-byte signature, deterministic four-cycle completion latency, and placeholder content `COREPH || digest || zero padding`.
\end{itemize}

\section{Control Semantics}
\begin{itemize}[leftmargin=*]
  \item `start`: launches a new adapter transaction when the wrapper is not busy.
  \item `clear\_status`: clears done and error state, and clears the exposed signature buffer when the wrapper is idle.
\end{itemize}

\section{Error Handling}
The wrapper currently reports an error if software attempts to start a new transaction while busy or touches an unsupported register region. A reserved engine error code is also defined for future adapter-detected failures, although no current deterministic adapter mode asserts it.

\section{Integration Note}
The wrapper contract is now explicitly independent of any specific downstream ML-DSA implementation. The next phase shall replace `CORE\_PLACEHOLDER` with ML-DSA-OSH integration behind the adapter boundary without changing the wrapper-visible register contract.
