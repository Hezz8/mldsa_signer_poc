\chapter{Detailed Register Map}

\section*{Purpose}
This chapter defines the stabilized AXI-Lite register map currently implemented by the software fake backend and the RTL wrapper stub.

\section{Register Summary}
\begin{longtable}{p{0.14\textwidth} p{0.19\textwidth} p{0.12\textwidth} p{0.45\textwidth}}
\toprule
Offset & Name & Access & Description \\
\midrule
0x000 & CONTROL & RW & Start and clear-status control bits. \\
0x004 & STATUS & RO & Idle, busy, done, and error status indicators. \\
0x008 & ERROR\_CODE & RO & Coarse error code for wrapper or backend-detected failure. \\
0x00C & SIG\_LENGTH & RO & Reported signature length in bytes. \\
0x010-0x04C & DIGEST[0:15] & RW & Sixteen 32-bit words holding the 64-byte input digest. \\
0x100-0x17C & SIG\_DATA[0:31] & RO & Thirty-two 32-bit words exposing the current 128-byte stub signature buffer. \\
\bottomrule
\end{longtable}

\section{Control Register Bits}
\begin{itemize}[leftmargin=*]
  \item Bit 0: `start`
  \item Bit 1: `clear\_status`
\end{itemize}

\section{Status Register Bits}
\begin{itemize}[leftmargin=*]
  \item Bit 0: `idle`
  \item Bit 1: `busy`
  \item Bit 2: `done`
  \item Bit 3: `error`
\end{itemize}

\section{Current Stub Semantics}
\begin{itemize}[leftmargin=*]
  \item Software shall fully populate the digest registers before asserting `start`.
  \item `start` shall transition the engine to busy when idle.
  \item The current stub implementation completes after a deterministic two-step progression.
  \item `clear\_status` clears done and error latches and clears the signature window when the engine is idle.
  \item The current fake signature occupies 128 bytes and is defined as `STUBSIG || digest || zero padding`.
\end{itemize}

\section{Current Error Codes}
\begin{itemize}[leftmargin=*]
  \item 0: no error
  \item 1: start requested while busy
  \item 2: unsupported register offset touched by the current stub implementation
\end{itemize}
